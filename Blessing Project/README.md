- Handle user events and interactions
- Create an engaging user experience

- Build an app with changing shapes, colors, and positions
- Implement proper event handling for user interactions

- Ensure variety and unpredictability in the experience

## My Coding Standards

### Comment Requirements
- **Extensive commenting:** Every few lines, if not every line
- **Explain the "why":** Not just what the code does, but why I chose this approach
- **Original design:** All onEvent, functions, traversals designed by me
- **External sources:** Only for specific syntax, properly cited
- **Justification:** Explain why each decision was made

## How I'm Using This Repository

### Step 1: Understanding the Requirements
I've created detailed guides for each assignment that explain:
- What I need to accomplish
- Why each requirement matters
- How to approach the problem
- What coding structures to demonstrate

### Step 2: Planning My Implementation
For each assignment, I:
- Read the requirements carefully
- Plan my approach before writing code
- Think about what would make a good user experience
- Consider how to demonstrate all the required coding structures

### Step 3: Writing My Code
I follow these principles:
- Start with a clear plan
- Add comments as I write code
- Test each feature as I implement it
- Make sure my code is readable and well-organized

### Step 4: Testing Everything
I use the test environment to:
- Verify all functionality works correctly
- Test edge cases and error conditions
- Make sure the user experience is smooth
- Confirm all coding structures are properly demonstrated

### Step 5: Transferring to code.org
Once everything works in the test environment:
- Copy my code to the code.org platform
- Test again in the actual environment
- Make any necessary adjustments
- Submit my assignments
